消防设计图纸 AI 标注规范说明（详细版 v3）

适用：地铁车站消防报建图（总平面图 + 站厅层平面图）的数据标注，用于训练 AI 并自动比对消防规范。 本版相对前版的三大改动：①去掉"文字类"冗余标注（改由程序从矢量直抽）②补齐缺失标签（闸机、安全出口）③新增防火分区划分专项规则。

0. 总则

  1. 优先用带文字层的矢量 PDF。文字相关数据（分区编号/面积/房间名/尺寸数字/属性文字）由程序直接抽取，标注团队不标（见第 8 节）。无文字层的图（如贝尔路、明海）才回退人工标文字。
  2. 标注团队的核心任务 = 标"图形符号"和"轮廓多边形"——这些没有文字、程序认不出，必须人工。
  3. 防火分区一律照图纸自带的"防火分区示意图"描，不得凭房间自行分区（见第 5 节）。
  4. 标签名 100% 英文 snake_case，与本规范附表完全一致，不得拼错、不得用中文、不得混用。
  5. 每批提交前用质检脚本 fire_anno_tool.py qc 自检，零 ERROR 才可交付。
  6. 交付格式 CVAT XML 1.1，保留多边形全部顶点、框宽高、属性值。

1. 标注范围总览

图纸 标注团队要标（本规范第 3、4 节） 程序直抽、团队不标（第 8 节）
站厅层 fire_compartment、public_area、commercial_shop、safety_exit、stair_escalator、gate、fire_door、fire_shutter、draft_curtain 分区编号/面积、房间名、各类尺寸数字、疏散距离值
总平面图 station_exit_ground、vent_group_ground、surrounding_building 建筑属性文字、风亭类别文字、防火间距尺寸数字

2. 通用标注规则

  • 几何类型：
    • 框(box)：图形符号对象（出口、楼梯、门、闸机等）。框要紧贴符号，不留多余空白、不切掉。
    • 多边形(polygon)：区域/轮廓（防火分区、商铺、周边建筑等）。顶点须落在真实边界线上。
    • 线(polyline)：仅在无文字层、需人工标尺寸线时使用（见第 8 节说明）。
  • 坐标精度：多边形在转角/节点处误差 ≤ ±2 像素（防火分区尤其严格，决定闭合判定）。
  • 不漏不重：同一对象只标一次；同类对象不得遗漏。
  • 标签语言：英文 snake_case（附表为准）；属性键也用英文，属性值可用中文枚举。

3. 站厅层 — 逐标签详细规则

3.1 fire_compartment（防火分区）— polygon ★重点

  • 画法：严格照图纸"防火分区示意图"，沿防火墙 / 特级防火卷帘 / 防火门勾勒完整闭合多边形。
  • 属性 zone_type（必填，三选一）：公共区 / 无人区 / 有人值守区。
  • 闭合要求：首尾顶点重合，转角误差 ≤2px，边界贴墙内中线；用防火卷帘分隔处，顶点须落在卷帘线上。
  • 关键约束（详见第 5 节）：一个分区通常含多个房间；泵房/厕所不单独成区。
  • 常见错误：❌一房一区　❌把消防泵房/污水泵房单独画成一个分区　❌边界没闭合/没贴防火墙。

3.2 public_area（站厅公共区）— polygon

  • 画法：付费区 + 非付费区合并后的公共活动空间整体边界。
  • 常见错误：❌漏掉非付费区　❌把设备区也圈进来。

3.3 commercial_shop（商铺）— polygon ★易错

  • 画法：只标公共区内真正的商铺，沿商铺边界画。
  • 常见错误：🔴把每个房间/隔间都标成商铺（历史批次出现 30 个商铺=房间数，严重错误）。规范公共区商铺 ≤3 个，正常一张图商铺很少。标之前先确认那确实是商业铺位，不是设备房。

3.4 safety_exit（安全出口）— box ★新增必标

  • 画法：框住"安全出口"图例/标识（疏散指示、EXIT 标志或楼梯/通道在公共区的出口位置）。
  • 用途：疏散个数与距离计算的起终点。
  • 常见错误：❌漏标（疏散计算就没有终点）。

3.5 stair_escalator（楼梯/扶梯）— box

  • 画法：框住站厅通向站台/出入口的楼梯、扶梯符号侧边范围。

3.6 gate（闸机）— box ★新增必标

  • 画法：框住闸机符号（AFC 进出站闸机阵列，逐个或逐排框）。
  • 常见错误：🔴整类遗漏（历次批次都没标，必须补）。

3.7 fire_door（防火门）— box

  • 画法：框住防火门符号，包含开启方向弧线。
  • 属性：
    • class（必填，三选一）：
      • 甲级：旁边写 "FM甲"
      • 乙级：旁边写 "FM乙"
      • unknown：图上没有 FM 字样（设备区房间间小门常见）。不要凭经验猜，QC 会汇总 unknown 清单交设计师/消防工程师批量补字。
    • swing_dir（必填）：顺着疏散方向 / 逆着疏散方向（看门扇弧线朝向）。
  • 常见错误：❌没把开启弧线框进去　❌属性漏填　❌看不到 FM 字样就漏标整扇门（应标 box，class 填 unknown）。

3.8 fire_shutter（防火卷帘）— box

  • 画法：框住特级防火卷帘符号/编号（如 FJLM3630）。
  • 用途：作为防火分区边界依据，分区多边形顶点须与其重合。

3.9 draft_curtain（挡烟垂壁）— box

  • 画法：框住挡烟垂壁标志线。

4. 总平面图 — 逐标签详细规则

4.1 station_exit_ground（车站地面出入口/疏散口）— polygon

  • 画法：A/B/C 出入口、地面垂梯、紧急疏散口暴露在地面的结构外墙投影面。

4.2 vent_group_ground（风亭/风亭组）— polygon

  • 画法：各号风亭/风亭组的地面结构外轮廓；高风亭标底座投影。
  • 属性（文字层能直读时由程序填，文字打散成曲线 / 复制不出来时由标注员对照图例肉眼录入）：
    • vent_function（必填，枚举）：新风 / 排风 / 活塞风 / 冷却塔风 / 排烟。规则引擎判"新风与排风/活塞风口距离 ≥10m"必须靠这个。
    • discharge_type（必填，枚举）：侧出 / 敞口 / 高风亭。括号里的"（侧出）/（敞口）"直接录入；高风亭看图上是否单列"高风亭"标注。
    • name（可选）：自由文本，如 2号风亭排风井 / 冷却塔风井，便于追溯。
    • area_m2（可选）：图上写明的风口面积（如"侧出风口面积 24m²"）。
  • 常见错误：❌只标多边形不填 vent_function，规则引擎无法判风口距离　❌把"侧出"塞进 vent_function（应在 discharge_type）。

4.3 surrounding_building（周边建筑）— polygon ★必标

  • 画法：图纸红线内外、影响防火间距的既有/规划建筑，沿外墙投影画（如"尚游公馆""上油松村民房""规划龙辉花园建筑"）。
  • 属性（文字层能直读时由程序填，文字打散成曲线 / 复制不出来时由标注员对照建筑旁文字录入；防火间距判定全靠这几个字段）：
    • name（必填）：建筑名称，自由文本，如 规划龙辉花园建筑 / 尚游公馆。
    • building_type（必填，枚举）：多层民用 / 高层民用 / 超高层民用 / 丙类厂房 / 丁戊类厂房 / 甲乙类厂房库房 / 加油加气加氢站 / 其他 / unknown。判档靠高度/层数：住宅 >27m、公建 >24m 为高层，>100m 为超高层。图上既无层数也无高度、无法判定时填 unknown（注意 unknown≠其他：其他指非民用非厂房的类别，unknown 指信息缺失待补），由设计师/消防工程师后补。住宅、商业、办公、酒店、医院、学校等都属民用建筑，按高度归多层/高层/超高层民用，不要填 其他。
    • fire_rating（必填，枚举）：一级 / 二级 / 三级 / 四级 / unknown。图上无此信息时填 unknown，由设计师/消防工程师后补。
    • floors（可选，整数）：层数，如 48（图上"住宅48F"）。
    • height_m（可选，数值）：建筑高度（米），便于区分高层（>24m）。
  • 常见错误：🔴整类遗漏（历次都缺，导致防火间距完全无法审查，必须补全）。❌只画多边形不填 building_type / fire_rating，防火间距规则取不到分支阈值。

重要：周边建筑的属性文字（"规划龙辉花园建筑/超高层居住建筑/耐火等级一级"那类）不要框文字、不要用 has_dimension 关联——直接把这三项录进 surrounding_building 多边形的属性里即可。复制不出来的就肉眼读。

5. 防火分区划分专项规则 ★最易出错

  1. 以"防火分区示意图"为唯一依据。图纸右上/右下角的防火分区示意图（按颜色分区+面积表）是标准答案，照着描，不得自行按房间划分。
  2. 一个防火分区 = 一片区域，含多个房间，由防火墙/防火卷帘整片围合，不是一房一区。
  3. 泵房/厕所"不计入面积"≠"独立成区"：消防水泵房、污水/废水泵房、厕所、盥洗等，面积可不计入所在分区，但它们仍属于相邻的某个分区，不单独画成一个分区。
  4. 新风/排风道与环控机房划为同一分区，不要单独切出去。
  5. 分区类型用 zone_type 标注：公共区 / 无人区 / 有人值守区。
  6. 面积红线（程序自动算，标注只需画准边界）：公共区每分区 ≤5000㎡；设备管理区每分区 ≤1500㎡（地下）。
  7. 最终分区合理性由有资质消防设计/审查人员确认；标注团队只负责"照示意图准确描出闭合边界"。

6. 属性字段规范（枚举）

属性键 适用标签 允许取值 必填
zone_type fire_compartment 公共区 / 无人区 / 有人值守区 是
class fire_door 甲级 / 乙级 / unknown（图上无 FM 字时） 是
swing_dir fire_door 顺着疏散方向 / 逆着疏散方向 是

area_m2 vent_group_ground 数值（风口面积，图上有则填） 否

⚠️ 属性键必须是上面这些英文键，不得出现 .、空键、中文键（历史批次出现 . 垃圾键，禁止）。

⚠️ 凡是 unknown 的字段（class / fire_rating / building_type 等）会被 QC 工具汇总成一份清单，交给设计单位/消防工程师批量补字。规则引擎遇到 unknown 不会误报合规/违规，会返回 review_required（待人工复核）。

7. 关系绑定（仅保留必要部分）

带文字层时尺寸值由程序直抽，99% 自动绑定靠就近原则，标注员不用动手。

只在两类情形下人工补关系：

7.1 has_dimension：对象 → 数字（仅当多个数字挤在一起、程序分不清谁是谁时）

  • 例：一扇防火门旁同时写了 1.05(洞口宽) / 0.9(净宽) / FM-1(编号) / 1.5h(耐火) → 标注员把"0.9"这个数字框作为子对象，连一条 has_dimension 到防火门 box
  • 适用主对象：fire_door box / 程序抽出来的尺寸线
  • 适用子对象：程序抽出来的数字框（不是文字框）

7.2 属性文字不要用 has_dimension

误用预警：周边建筑的"耐火等级一级 / 超高层居住建筑"、风亭的"侧出 / 敞口"这种属性文字，不用关系绑定，直接录进对象的属性键（fire_rating / building_type / vent_function / discharge_type），无论文字能不能复制都一样。

7.3 何时不用做任何关系

绝大多数情况：数字唯一、对象唯一、就近能配对。标注员什么都不用做，程序自动配。

8. 不需要标注的内容（程序从矢量直抽）

以下数据带文字层时一律由程序直接抽取，标注团队不标，避免重复劳动和人工录入错误：

数据 原因
防火分区编号/面积/类型文字 文字层直读（如"防火分区一（公共区）3861.71㎡"）
房间名称 文字层直读
门宽/走道宽/楼梯宽等尺寸数字 文字层直读
疏散距离数值 文字层直读
周边建筑属性（耐火等级/层数/类别） 文字层直读
风亭类别文字 文字层直读
防火间距尺寸数字 文字层直读
墙线/尺寸线几何坐标 矢量几何直抽

例外：无文字层的图纸（文字被打散成曲线），上述文字类回退人工标注（box+text_content），并视为"为 OCR 模型造训练数据"。

8.1 30 秒自检：这张图有没有文字层？

拿到一张 PDF 图纸先做这个判断：

  1. 用 PDF 阅读器（Adobe / Foxit / 浏览器）打开
  2. 鼠标拖选图纸上任意一段文字（分区编号、房间名、尺寸数字都行）
  3. Ctrl+C 复制到记事本看看：
    • ✅ 能选中、复制出原文字 → 有文字层，所有数字/属性文字程序直抽，标注员不用录任何文字
    • ❌ 选不中 / 复制出来是空白 / 是方块乱码 → 文字打散了，按"无文字层"流程处理

8.2 无文字层时怎么标？分两种情况

文字类型 处理方式
对象的属性文字（建筑耐火等级/类型、风亭功能/出风方式、防火门 FM 级别） 直接录进对象的属性键（fire_rating / building_type / vent_function / discharge_type / class）。不要单独框文字、不要做关系绑定。
散落的数字（尺寸、面积、疏散距离） 框 box + 填 text_content 属性，相当于人工帮 OCR 补真值。这些标注会被收集为后续 OCR 模型的训练数据。

9. 质量验收（DoD）

  1. 格式自检：python fire_anno_tool.py qc 标注.xml —— 零 ERROR。
  2. 完整性：必现类零缺失（总平面图必有 surrounding_building；站厅层必有 ≥2 个 fire_compartment 含设备区、safety_exit、gate）。
  3. 几何：多边形闭合、转角 ≤2px、框紧贴对象。
  4. 金标准：与专家精标的样图比对，框 IoU ≥0.9、标签一致率 ≥95%。
  5. 人工抽检：错误率 <5%（稀有/高危类全检，常见类抽 10–20%）。

10. 常见错误红线清单（提交前自查）

  • 🔴 周边建筑（surrounding_building）漏标 → 防火间距无法审
  • 🔴 设备区防火分区漏标（只标了公共区）
  • 🔴 闸机（gate）整类漏标
  • 🔴 安全出口（safety_exit）漏标
  • 🔴 把房间/隔间误标成商铺（商铺数 = 房间数是典型信号）
  • 🔴 把消防泵房/污水泵房单独画成一个防火分区
  • 🟠 标签用中文 / 拼错 / 属性键出现 . 或空键
  • 🟠 防火分区没闭合、没贴防火墙
  • 🟠 防火门没框进开启弧线、class/swing_dir 漏填
  • 🟠 周边建筑只画了多边形，没填 building_type / fire_rating（防火间距规则没法跑）
  • 🟠 风亭没填 vent_function（"新风与排风口距离"规则没法跑）
  • 🟠 把"侧出/敞口"塞进 vent_function（应在 discharge_type）
  • 🟠 看不到 FM 字样的防火门漏标整扇门（应标 box，class 填 unknown）
  • 🟠 用 has_dimension 关联属性文字（应直接录进对象属性键）

11. 交付格式

  • 文件：CVAT XML 1.1（每张图一个 annotations.xml + 对应底图）。
  • 底图：高 DPI（≥200）渲染，保证闸机/小符号清晰；大图按 ~1024² 切片。
  • 命名：与车站、图纸类型对应，便于追溯。
  • 标签/属性键：全英文 snake_case，与本规范附表一致。

本规范为 MVP 阶段标注依据，随项目推进迭代。防火分区等专业判定最终以注册消防工程师/审查意见为准。
