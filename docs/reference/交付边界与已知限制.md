# 交付边界与已知限制

本文件界定本系统**当前版本**的能力范围、已知限制、外部依赖与适用前提,供使用方与接手方对齐预期。

> **定位**:本系统是"AI 预审 + 人工复核"工具,输出为**预审意见**,所有结论须经人工确认,不作为终审放行依据。

为便于非技术读者理解,文中首次出现的专业术语均附简要说明。

---

## 一、系统概述

系统接收一张消防设计图(PDF),依次完成四步:

1. **识别**:定位图上的防火分区、安全出口、门、楼梯、闸机、商铺等对象。
2. **量测**:计算防火分区面积、门与走道净宽、公共区至最近安全出口的疏散距离。
3. **比对**:依据内置《消防规范》知识库的量化条款逐项判定是否合规。
4. **输出**:在原图标注不合规位置,并生成可交付的 Word 审查意见表。

涉及的关键概念:

- **矢量 PDF**:由 CAD 等制图软件直接导出的 PDF,其线条与文字为可解析的矢量数据(可选中文字、缩放不失真),区别于扫描件/图片型 PDF。本系统的自动识别与比例尺标定**依赖矢量 PDF**。
- **比例尺标定**:确定"图上 1 像素对应现实多少米"的过程。系统自动读取图纸文字层中的比例标注(如 `1:200`)换算。
- **CVAT 标注**:由人工在标注工具中框选对象、赋标签产出的"真值数据"(标准答案)。
- **YOLO 模型**:目标识别神经网络模型,经训练后可自动识别图上对象,用于替代人工标注。

## 二、支持范围

### 2.1 输入
单张矢量 PDF 图纸(站厅层平面图或总平面图)。

### 2.2 识别(两种通路,二选一)
- **方式①——CVAT 真值标注**:识别准确率 100%,适用于正式交付与演示,前提是已完成人工标注。
- **方式②——YOLO 模型自动识别**:无需人工、速度快,准确率取决于模型(见 §3.2);权重随仓库交付。

可识别对象共 **16 类**:防火分区、公共区、安全出口、楼梯/扶梯、闸机、防火门、商铺、挡烟垂壁、防火卷帘、地面风亭、周边建筑等。

### 2.3 量测与结构化
- 比例尺自动标定(从矢量文字层读取 `1:N`)。
- 几何量测:防火分区面积、门/走道/楼梯净宽、疏散距离。
- 面积兜底:矢量层缺面积数据时,以 OCR(光学字符识别,从图像中读取文字数字)补充。

### 2.4 规则比对
内置 **37 条规则,覆盖 8 类**:防火分区面积、疏散宽度、安全出口、防火门窗卷帘、商铺、安全疏散、防火间距、防排烟。规则为确定性条件判断(非 AI 推断),**每条结论均可追溯至具体规范条文与数值**,结果可解释、可复核。

### 2.5 疏散路径
自动计算公共区内最不利点至最近安全出口的疏散距离,与规范限值(一般 50 m)比较。

### 2.6 输出物(5 项)
标注图(标记不合规位置)、规则评估结果(JSON)、结构化数据(JSON)、Markdown 简报、**Word 审查意见表(可交付)**。

### 2.7 桌面客户端
图形界面(拖拽即用),含四个页面:数据准备、端到端预审、规则库、YOLO 训练。

## 三、已知限制

### 3.1 门净宽判定存在系统性偏差(优先级最高)

现行算法:取门洞尺寸标注线的像素长度 × 比例尺得到门洞宽,再统一扣减 0.15 m(估算门框占用)作为**净宽**(门扣除门框后的实际通行宽度),与规范限值 0.9 m 比较。

该方法在阈值附近不稳定:标准单扇门(门洞标定 1.0 m,实测约 1.04 m)扣减后为 0.89 m,因约 1 cm 之差被判不合格。同一图纸的标准门会集中落在阈值下方,呈现非真实的批量不合格(欧洲城站实测:32 扇门中 22 扇如此)。

- **影响**:门净宽规则(`WIDTH-DOOR-EVAC-001`)的不合格结果可靠性低,**须人工复核后采信**;明显偏窄(如 0.7 m)的门仍能正确识别。
- **现状**:本版未做修改,在此如实告知。根因分析与正确解法见 §六。

### 3.2 YOLO 自动识别对稀有类别召回不足

模型基于 9 个站、十余张图训练,样本量有限。**高频类别识别良好**(闸机、风亭、防火门的 mAP 均在 0.9 以上;mAP 为衡量识别准确率的综合指标,取值 0–1,越高越好),**低频类别基本无法识别**(安全出口、商铺因训练样本不足 20 个,mAP 近 0)。

- **影响**:采用方式②(YOLO)时,安全出口、商铺等可能漏检。正式交付建议采用方式①(CVAT 真值)规避;提升模型需补充更多站点的标注数据重新训练。

### 3.3 结果质量受标注质量制约

采用人工标注通路时,**标注质量直接决定结果准确性**。已发现个别站点(侨社、明海)的门宽标注线长度仅为实际门宽的约一半,导致门宽量测整体失真。接收第三方标注时不应默认其准确,应先经质检。

### 3.4 区域细分不完整

规范要求站厅层区分公共区、设备区、付费区、非付费区四类,当前仅识别前三类,**付费区/非付费区尚未拆分**。需扩充标注规范并由标注方补充标注。

### 3.5 比例尺标定的适用条件

系统依赖图纸文字层中的比例标注(如 `1:200`)。以下情形会失效或出错:
- **扫描件/图片型 PDF**(无文字层):无法自动标定,须手动指定比例尺。
- **同图含多个比例**(如主图 `1:200` 与示意图 `1:1000` 并存):系统仅取其一,存在误选风险;目前依据整体尺寸合理性大体可选对,但**未实现按视口分区标定**——主图与详图比例混排时,详图区域尺寸将整体失真。

### 3.6 疏散距离为栅格近似

疏散路径在降采样栅格(将公共区简化为粗网格)上计算,**未扣除墙体、柱子等障碍**(默认公共区内部可直线通行),结果为合理近似值而非精确值。且需图中同时具备公共区范围、比例尺、安全出口三项,缺一即跳过该项计算。

### 3.7 产物命名与目录管理风险

产物默认以"站名-图类型"命名;当无法从源文件名识别站名时,退化为不含站名的通用名。此时若人工归档至站点目录,易发生错配(已发生实例:侨社站目录中存放了欧洲城站的预审结果)。**建议核对产物中记录的源图名与所在目录是否一致**。此外,同一图纸重复运行会覆盖既有产物,需保留时应先更改输出路径。

### 3.8 其他边界

- 图纸标注的面积/尺寸数值为**设计方声称值**,系统直接读取,**未做几何复核**;若图纸本身标注有误,系统暂不能发现。
- 单个 PDF 仅处理**第一页**,多页图纸不自动拆分。
- OCR 当前仅安装英文/数字识别包,未安装中文识别包。
- 规则覆盖**可由几何尺寸判定**的条款;构造做法、材料、设备联动等无法从图面量测的条款不在覆盖范围。

## 四、外部资源依赖

- **Python 依赖库**:于项目根目录执行 `pip install -r requirements.txt` 一次安装(含训练所需的 `ultralytics`)。
- **Tesseract OCR**(系统级软件,需单独安装):Windows 安装 UB-Mannheim 版本至默认目录,系统自动探测;当前仅含英文/数字识别。
- **NVIDIA 显卡 + CUDA**(仅训练时需要):本机为 RTX 5060,已配置。RTX 50 系训练须使用特定参数(见训练手册),否则会中途失败;**仅做预审、不训练时,无显卡的普通计算机亦可运行(速度较慢)**。
- **图纸与标注数据**(保密,不纳入代码仓库,需向项目负责人获取):9 个站点的矢量 PDF 及对应人工标注。
- **随仓库交付、无需另行准备**:训练完成的 YOLO 权重(`models/` 目录)、37 条规则知识库(`rules/`)。

## 五、适用前提

满足以下条件方可获得可靠结果:

1. 输入为**矢量 PDF**,且文字层含比例标注(否则须手动指定比例尺)。
2. 采用人工标注时,标注须符合规范(尺寸线完整横跨被测对象、标签合规),见 [标注规范说明_详细版.md](标注规范说明_详细版.md)。
3. 单次预审针对**单站单图**(站厅层或总平面图),且全图比例统一。
4. 规则阈值取自内置知识库,**最终判定须经人工复核**。

## 六、后续改进路线(按优先级)

| 优先级 | 事项 | 说明 |
|---|---|---|
| 高 | 门净宽判定改造 | 解析图纸门型号代码(如 `FM甲1023` 对应门洞 1.0 m)取标定宽度,或读取门窗表,替代"像素量测 − 0.15 m";几何量测降为交叉校验。根治 §3.1。 |
| 高 | 拆分付费区/非付费区 | 扩充标注规范并由标注方补标,补齐区域四分类。 |
| 中 | 接入门窗表 | 门窗统计表为门尺寸、防火等级的权威来源,作为门宽判定基准。 |
| 中 | YOLO 增量训练 | 补充 8–10 个站点样本,提升安全出口、商铺等低频类别识别率,见 [继续训练手册](../guides/yolo_incremental_training.html)。 |
| 中 | 比例尺按视口分区标定 | 解决同图多比例混排问题。 |
| 低 | 疏散路径扣除障碍 | 扣除墙、柱并提高栅格精度。 |
| 低 | 增加几何复核 | 重算面积与设计声称值交叉校验,识别标注笔误。 |
| 低 | 产物防覆盖与防错配 | 输出目录化;校验产物源图名与归档目录一致性。 |
