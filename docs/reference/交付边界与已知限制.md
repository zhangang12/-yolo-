## 一、支持什么(当前可用)

### 1.1 输入
- **矢量 PDF 图纸**:站厅层平面图、总平面图。要求 PDF **带文字层**(能选中文字),用于比例尺自动标定。
- **两条识别通路二选一**:
  - 方式①:已有 **CVAT 真值标注 XML**(识别 100% 准,推荐用于验证/交付)。
  - 方式②:**YOLO 模型权重** `models/fire_seg_gpu/best.pt`(切片推理 → 自动识别,零人工,精度受模型限制)。

### 1.2 识别(16 类)
防火分区、公共区、安全出口、楼梯/扶梯、闸机、防火门、商铺、挡烟垂壁、防火卷帘、风亭组、周边建筑等。

### 1.3 结构化与量测
- 比例尺自动标定:从 PDF 文字层读 `1:N` → m/px。
- 几何量测:防火分区面积(像素多边形×scale²)、门/走道/楼梯净宽(尺寸线×scale)、疏散距离(折线×scale)。
- 栅格 OCR 兜底:矢量层缺面积时,几何匹配补防火分区 `area_m2_design`。

### 1.4 规则比对(37 条 / 8 大类,确定性 if-else,可追溯)
| 类别 | 条数 | 类别 | 条数 |
|---|---|---|---|
| 防火分区面积 | 8 | 商铺 | 4 |
| 疏散宽度 | 7 | 安全疏散 | 3 |
| 安全出口 | 5 | 防火间距 | 3 |
| 防火门窗卷帘 | 5 | 防排烟 | 2 |

### 1.5 疏散路径
公共区栅格化 + 多源 Dijkstra,算"任一点 → 最近出口"最远疏散距离,喂 `EVAC-DIST-ANY-001`。

### 1.6 输出(5 个产物)
标注图 `_e2e_fail.jpg`(红框圈违规)、规则评估 `_findings.json`、结构化 `_structured.json`、Markdown 简报 `_report.md`、**Word 审查意见表 `_审查意见表.docx`**(可交付)。

### 1.7 桌面客户端(PySide6,4 页)
数据准备 / 端到端预审 / 规则库 / YOLO 训练。

## 二、不支持 / 已知限制(使用方必须知晓)

### 2.1 识别与标注精度
- **YOLO 稀有类 mAP=0**:`safety_exit` / `commercial_shop` 训练实例 <20,基本识别不出;`gate`/`vent`/`fire_door` 可用(0.9+)。交付走 CVAT 真值通路可规避。
- **付费区/非付费区未拆**:任务原文第 1 项 4 类只识别了 3 类(schema 缺细分)。
- **标注质量随站点波动**:个别站(如侨社、明海)的 `width_dimension_line` 被画成实际门宽的约一半(中位 ~19px vs 正常 ~41px),换算出的门净宽全部异常偏小、批量误判。**标注质量直接决定结果可信度**。

### 2.2 门净宽判定(重要,易误判)⚠️
- 当前算法:`净宽 = 洞口尺寸线像素长 × scale − 0.15m`,**纯几何 + 写死扣减常数**。
- 问题:标准单扇门(洞口 1000mm,量得 ~1.04m)扣 0.15 = **0.89m**,差 1cm 误判 <0.9m;一张图标准门会**集体落在阈值正下方**(实测欧洲城 22/32 门)。这是扣减常数 + ±1px 像素误差造成的伪失败,不是真实不合格。
- **当前状态**:暂未修改,门宽仍按"洞口×scale−0.15"判定,**会出现上述批量误判**(本版如实告知,使用时门宽 FAIL 项需人工复核)。
- **建议缓解(待跟进)**:规则引擎加**阈值容差带**(`review_tolerance`),净宽落在阈值下方容差内时判 REVIEW 而非硬 FAIL。
- **正确做法(待跟进)**:图纸每扇门旁有**门型号代码**(`FM甲1023`→洞口 1000mm),应解析代码取标定宽度 + 正确的洞口→净宽口径(或读门窗表),把几何降级为交叉校验。详见下文 §五 待跟进事项。

### 2.3 比例尺标定
- **依赖 PDF 文字层有 `1:N`**。栅格 PDF(无文字层,如明海/贝尔路)自动标定失败,需手填 `--scale`。
- **图纸含多个比例标注时**(如同一张图既有 `1:200` 又有 `1:1000` 详图/示意图),程序取其一,可能选错;目前按整体物理尺寸合理性大体可选对,但**未做按视口分区标定**(主图 1:200、详图 1:50 混排会让详图尺寸全错)。

### 2.4 疏散路径(栅格近似)
- downsample=20(1 cell≈20px)栅格上跑 Dijkstra,误差约 ±1 cell。
- **未扣除墙体/柱子等障碍**(public_area 内部默认全可通行),算的是直线可达栅格距离。
- 依赖 ① public_area(或 zone=公共区的 fire_compartment)多边形 ② 比例尺 ③ safety_exit 出口点,三者缺一则跳过。

### 2.5 数据/产物管理
- **产物按 `站名-图类型` 命名;站名识别失败时兜底为无前缀的 `站厅层_*`**,此时若手动放入某站目录,易发生**张冠李戴**(实测侨社站目录里曾放着欧洲城站的预审结果)。建议核对产物里的源图名与目标目录是否一致。
- **同 PDF 重跑覆盖 `e2e_out/` 旧产物**(同 stem)。

### 2.6 其他
- 设计院面积/尺寸为**声称值**:OCR/矢量直读取的是图上写的数字,**未做几何复核**(几何精提取校验器未实现)。
- 只处理 PDF **第 0 页**;多页/索引图不自动拆分。
- OCR 只装了 **eng**(英文/数字),未装 `chi_sim`;栅格 OCR 兜底只采带小数的数字。
- **门窗表(门窗统计表)未解析**:最权威的门净宽/防火等级来源未接入。
- 规则覆盖**几何可判**条文;构造做法、材料、设备联动等超几何识别范围的条文不在覆盖内。

## 三、外部资源依赖

### 3.1 软件 / 库(`requirements.txt`)
`pymupdf` · `opencv-python` · `numpy` · `pytesseract` · `Pillow` · `shapely` · `PySide6` · `python-docx`;训练另需 `ultralytics`。

### 3.2 系统级
- **Tesseract OCR**(系统安装,非 pip):Windows 装 UB-Mannheim 版到默认路径,项目自动探测(`tools/tesseract_init.py`)。当前只有 `eng`;要识别中文需另装 `chi_sim`。
- **GPU / CUDA(仅训练)**:RTX 50 系(Blackwell)必须 `--no-amp --workers 0 --batch 4`,否则训练崩;CPU 可跑只是慢。推理/预审 CPU 即可。

### 3.3 数据资产(保密,不入 git,向项目负责人索取)
- `data/` 下各站**矢量 PDF 图纸** + **CVAT 真值标注**(`anno_hall` 站厅层 / `anno_site` 总平面图,各 9 站)。
- 训练数据集 `dataset/`(由 CVAT 标注切片生成)。

### 3.4 随仓库交付(已入库)
- **YOLO 权重** `models/fire_seg_gpu/best.pt`(77MB)+ 训练报告。
- **规则知识库** `rules/rules.json`(37 条)+ `rules/schema.md`。

## 四、前置假设(满足才能得到可信结果)
1. 输入是**矢量 PDF**且文字层含 `1:N` 比例(否则需手填 scale)。
2. 标注遵循 [label_schema.md](label_schema.md) / [标注规范说明_详细版.md](标注规范说明_详细版.md):尺寸线完整横跨被测对象、标签/属性合规。
3. 一次预审针对**单站的单张图**(站厅层或总平面图);整体比例尺一致。
4. 规则阈值取自内置知识库,**最终判定需人工复核**(本系统定位为"AI 预审 + 人工复核",非自动放行)。
