# 消防设计图纸 AI 标注规范说明（详细版 v3）

> 适用：地铁车站消防报建图（总平面图 + 站厅层平面图）的数据标注，用于训练 AI 并自动比对消防规范。
> 本版相对前版的三大改动：①去掉"文字类"冗余标注（改由程序从矢量直抽）②补齐缺失标签（闸机、安全出口）③新增防火分区划分专项规则。

---

## 0. 总则

1. **优先用带文字层的矢量 PDF**。文字相关数据（分区编号/面积/房间名/尺寸数字/属性文字）**由程序直接抽取，标注团队不标**（见第 8 节）。无文字层的图（如贝尔路、明海）才回退人工标文字。
2. **标注团队的核心任务 = 标"图形符号"和"轮廓多边形"**——这些没有文字、程序认不出，必须人工。
3. **防火分区一律照图纸自带的"防火分区示意图"描**，不得凭房间自行分区（见第 5 节）。
4. **标签名 100% 英文 snake_case**，与本规范附表完全一致，不得拼错、不得用中文、不得混用。
5. 每批提交前用质检脚本 `fire_anno_tool.py qc` 自检，零 ERROR 才可交付。
6. 交付格式 **CVAT XML 1.1**，保留多边形全部顶点、框宽高、属性值。

---

## 1. 标注范围总览

| 图纸 | 标注团队**要标**（本规范第 3、4 节） | 程序**直抽**、团队**不标**（第 8 节） |
|---|---|---|
| 站厅层 | fire_compartment、public_area、commercial_shop、safety_exit、stair_escalator、gate、fire_door、fire_shutter、draft_curtain | 分区编号/面积、房间名、各类尺寸数字、疏散距离值 |
| 总平面图 | station_exit_ground、vent_group_ground、surrounding_building | 建筑属性文字、风亭类别文字、防火间距尺寸数字 |

---

## 2. 通用标注规则

- **几何类型**：
  - `框(box)`：图形符号对象（出口、楼梯、门、闸机等）。框要**紧贴符号**，不留多余空白、不切掉。
  - `多边形(polygon)`：区域/轮廓（防火分区、商铺、周边建筑等）。顶点须落在真实边界线上。
  - `线(polyline)`：仅在无文字层、需人工标尺寸线时使用（见第 8 节说明）。
- **坐标精度**：多边形在转角/节点处误差 **≤ ±2 像素**（防火分区尤其严格，决定闭合判定）。
- **不漏不重**：同一对象只标一次；同类对象不得遗漏。
- **标签语言**：英文 snake_case（附表为准）；属性键也用英文，属性值可用中文枚举。

---

## 3. 站厅层 — 逐标签详细规则

### 3.1 `fire_compartment`（防火分区）— polygon ★重点
- **画法**：严格照图纸"防火分区示意图"，沿**防火墙 / 特级防火卷帘 / 防火门**勾勒**完整闭合**多边形。
- **属性 `zone_type`**（必填，三选一）：`公共区` / `无人区` / `有人值守区`。
- **闭合要求**：首尾顶点重合，转角误差 ≤2px，边界贴墙内中线；用防火卷帘分隔处，顶点须落在卷帘线上。
- **关键约束**（详见第 5 节）：一个分区**通常含多个房间**；泵房/厕所**不单独成区**。
- **常见错误**：❌一房一区　❌把消防泵房/污水泵房单独画成一个分区　❌边界没闭合/没贴防火墙。

### 3.2 `public_area`（站厅公共区）— polygon
- **画法**：付费区 + 非付费区合并后的公共活动空间整体边界。
- **常见错误**：❌漏掉非付费区　❌把设备区也圈进来。

### 3.3 `commercial_shop`（商铺）— polygon ★易错
- **画法**：**只标公共区内真正的商铺**，沿商铺边界画。
- **常见错误**：🔴**把每个房间/隔间都标成商铺**（历史批次出现 30 个商铺=房间数，严重错误）。规范公共区商铺 ≤3 个，正常一张图商铺很少。**标之前先确认那确实是商业铺位，不是设备房。**

### 3.4 `safety_exit`（安全出口）— box ★新增必标
- **画法**：框住"安全出口"图例/标识（疏散指示、EXIT 标志或楼梯/通道在公共区的出口位置）。
- **用途**：疏散个数与距离计算的起终点。
- **常见错误**：❌漏标（疏散计算就没有终点）。

### 3.5 `stair_escalator`（楼梯/扶梯）— box
- **画法**：框住站厅通向站台/出入口的楼梯、扶梯符号侧边范围。

### 3.6 `gate`（闸机）— box ★新增必标
- **画法**：框住闸机符号（AFC 进出站闸机阵列，逐个或逐排框）。
- **常见错误**：🔴**整类遗漏**（历次批次都没标，必须补）。

### 3.7 `fire_door`（防火门）— box
- **画法**：框住防火门符号，**包含开启方向弧线**。
- **属性**：
  - `class`（必填）：`甲级` / `乙级`（可从旁边"FM甲/FM乙"文字判断）。
  - `swing_dir`（必填）：`顺着疏散方向` / `逆着疏散方向`（看门扇弧线朝向）。
- **常见错误**：❌没把开启弧线框进去　❌属性漏填。

### 3.8 `fire_shutter`（防火卷帘）— box
- **画法**：框住特级防火卷帘符号/编号（如 FJLM3630）。
- **用途**：作为防火分区边界依据，分区多边形顶点须与其重合。

### 3.9 `draft_curtain`（挡烟垂壁）— box
- **画法**：框住挡烟垂壁标志线。

## 4. 总平面图 — 逐标签详细规则

### 4.1 `station_exit_ground`（车站地面出入口/疏散口）— polygon
- **画法**：A/B/C 出入口、地面垂梯、紧急疏散口暴露在地面的结构外墙投影面。

### 4.2 `vent_group_ground`（风亭/风亭组）— polygon
- **画法**：各号风亭/风亭组的地面结构外轮廓；高风亭标底座投影。

### 4.3 `surrounding_building`（周边建筑）— polygon ★必标
- **画法**：图纸红线内外、影响防火间距的既有/规划建筑，沿外墙投影画（如"尚游公馆""上油松村民房"）。
- **常见错误**：🔴**整类遗漏**（历次都缺，导致防火间距完全无法审查，必须补全）。

> 注：周边建筑的**耐火等级/层数/类别**文字、防火间距**尺寸数字**——由程序从文字层直抽，**不用框文字**（见第 8 节）；无文字层时才人工标。

---

## 5. 防火分区划分专项规则 ★最易出错

1. **以"防火分区示意图"为唯一依据**。图纸右上/右下角的防火分区示意图（按颜色分区+面积表）是标准答案，**照着描**，不得自行按房间划分。
2. **一个防火分区 = 一片区域，含多个房间**，由防火墙/防火卷帘整片围合，**不是一房一区**。
3. **泵房/厕所"不计入面积"≠"独立成区"**：消防水泵房、污水/废水泵房、厕所、盥洗等，面积可不计入所在分区，但**它们仍属于相邻的某个分区，不单独画成一个分区**。
4. **新风/排风道与环控机房划为同一分区**，不要单独切出去。
5. **分区类型**用 `zone_type` 标注：公共区 / 无人区 / 有人值守区。
6. **面积红线**（程序自动算，标注只需画准边界）：公共区每分区 ≤5000㎡；设备管理区每分区 ≤1500㎡（地下）。
7. 最终分区合理性由**有资质消防设计/审查人员**确认；标注团队只负责"照示意图准确描出闭合边界"。

## 6. 属性字段规范（枚举）

| 属性键 | 适用标签 | 允许取值 | 必填 |
|---|---|---|---|
| `zone_type` | fire_compartment | 公共区 / 无人区 / 有人值守区 | 是 |
| `class` | fire_door | 甲级 / 乙级 | 是 |
| `swing_dir` | fire_door | 顺着疏散方向 / 逆着疏散方向 | 是 |
| `text_content` | （仅无文字层时的文字框） | 原文字符串 | 视情况 |

> ⚠️ 属性键必须是上面这些英文键，**不得出现 `.`、空键、中文键**（历史批次出现 `.` 垃圾键，禁止）。

---

## 7. 关系绑定（仅保留必要部分）

带文字层时尺寸值由程序直抽，一般不需人工绑定。**仅在一个对象旁有多个数字、易混淆时**，对关键尺寸做关联：
- 主对象：防火门 box / 尺寸线　→　子对象：对应数字
- 关联键：`has_dimension`

---

## 8. 不需要标注的内容（程序从矢量直抽）

以下数据**带文字层时一律由程序直接抽取，标注团队不标**，避免重复劳动和人工录入错误：

| 数据 | 原因 |
|---|---|
| 防火分区编号/面积/类型文字 | 文字层直读（如"防火分区一（公共区）3861.71㎡"） |
| 房间名称 | 文字层直读 |
| 门宽/走道宽/楼梯宽等尺寸数字 | 文字层直读 |
| 疏散距离数值 | 文字层直读 |
| 周边建筑属性（耐火等级/层数/类别） | 文字层直读 |
| 风亭类别文字 | 文字层直读 |
| 防火间距尺寸数字 | 文字层直读 |
| 墙线/尺寸线几何坐标 | 矢量几何直抽 |

> **例外**：无文字层的图纸（文字被打散成曲线），上述文字类回退人工标注（box+text_content），并视为"为 OCR 模型造训练数据"。

## 9. 质量验收（DoD）

1. **格式自检**：`python fire_anno_tool.py qc 标注.xml` —— 零 ERROR。
2. **完整性**：必现类零缺失（总平面图必有 surrounding_building；站厅层必有 ≥2 个 fire_compartment 含设备区、safety_exit、gate）。
3. **几何**：多边形闭合、转角 ≤2px、框紧贴对象。
4. **金标准**：与专家精标的样图比对，框 IoU ≥0.9、标签一致率 ≥95%。
5. **人工抽检**：错误率 <5%（稀有/高危类全检，常见类抽 10–20%）。

---

## 10. 常见错误红线清单（提交前自查）

- 🔴 周边建筑（surrounding_building）漏标 → 防火间距无法审
- 🔴 设备区防火分区漏标（只标了公共区）
- 🔴 闸机（gate）整类漏标
- 🔴 安全出口（safety_exit）漏标
- 🔴 把房间/隔间误标成商铺（商铺数 = 房间数是典型信号）
- 🔴 把消防泵房/污水泵房单独画成一个防火分区
- 🟠 标签用中文 / 拼错 / 属性键出现 `.` 或空键
- 🟠 防火分区没闭合、没贴防火墙
- 🟠 防火门没框进开启弧线、class/swing_dir 漏填

---

## 11. 交付格式

- 文件：CVAT XML 1.1（每张图一个 annotations.xml + 对应底图）。
- 底图：高 DPI（≥200）渲染，保证闸机/小符号清晰；大图按 ~1024² 切片。
- 命名：与车站、图纸类型对应，便于追溯。
- 标签/属性键：全英文 snake_case，与本规范附表一致。
