🔥 消防图纸 AI 预审 · 客户端使用文档

本文档面向实际使用人员(预审员、标注员、技术管理),覆盖客户端 4 个页面的完整用法。 客户端是个 PySide6 桌面程序,把命令行能力包成图形界面 — 选文件、点按钮即可。

1. 启动与界面布局

1.1 启动方式

# 项目根目录(-yolo-)下:
pip install -r requirements.txt
python app/main.py

首次启动:

  • 依赖包大约 1–2 GB(主要是 PySide6、PyTorch、ultralytics)
  • GPU 推荐:RTX 系列显卡可大幅加速训练(本项目在 RTX 5060 上验证过)
  • 纯 CPU 也能跑,只是训练慢 8-15 倍

1.2 系统级依赖 — Tesseract OCR 部分功能需要

Python 包(requirements.txt)装完还不够:OCR 是系统级程序, Python 只是调用它的封装。没装 tesseract 时,客户端"端到端预审"和"数据准备 → 预标注(勾选 OCR)" 会报这个错:

❌ 未检测到 tesseract(OCR 引擎)。第①阶段读数字需要它。
   安装:Windows 装 UB-Mannheim/tesseract 安装包并加入 PATH;
        macOS `brew install tesseract`;Ubuntu `sudo apt install tesseract-ocr`。
系统安装方式中文识别附加
Windows从 UB-Mannheim/tesseract 下载安装包,安装时把 PATH 勾上(或安装后手动把 C:\Program Files\Tesseract-OCR 加入系统 PATH)安装时勾"Additional script data" → Chinese Simplified
macOSbrew install tesseractbrew install tesseract-lang
Ubuntu / Debiansudo apt install tesseract-ocrsudo apt install tesseract-ocr-chi-sim

验证安装成功:任意终端执行 tesseract --version,能输出版本号即可。客户端无需重启, 下次点"开始预审"时会自动检测。

不想配 PATH?项目会自动找。
脚本会在 PATH 找不到时,自动探测 Windows / macOS / Linux 的常见安装目录(如 C:\Program Files\Tesseract-OCR\、/opt/homebrew/bin/、 /usr/bin/),探测到就直接用,无需手工改环境变量。 所以只要 Tesseract 装在默认目录就能跑;PATH 配好只是让命令行 tesseract 也能用而已。 背后逻辑:tools/tesseract_init.py 的 ensure_tesseract()。
什么时候必须装:
  • ✅ 端到端预审时,图纸没有矢量文字层(扫描件 / 老 PDF)— OCR 是唯一退路
  • ✅ 数据准备 → 预标注勾"启用 OCR"做"机器先标"
  • ❌ 矢量 PDF 端到端审查 — 文字层直读优先,不靠 OCR;不装也能跑(只是不读图上数字时降级为 review_required)

1.3 主界面

左侧蓝色侧边栏:4 个导航按钮 — 数据准备 / 端到端预审 / 规则库 / YOLO 训练

右侧大区域:当前选中页面的工作区,通常分两部分:

  • 左侧控制卡片(宽 400 px):填参数、点按钮
  • 右侧分屏(上:图片预览,下:实时日志)
所有任务都用子进程跑 — 不会卡死界面、可以随时点"停止",日志会实时滚动出来。

2. 页面一:数据准备 三件套

把原始图纸或标注文件,加工成可以用于 AI 训练 / 质检的数据。

2.1 预标注 prelabel

用途:在图纸上自动框出文字、尺寸线、防火分区等元素并预先分类,人工只需复查纠正, 不用从白纸开始一个个画。能减少标注员 60–80% 的工作量。

输入:

字段说明
图纸 PDF必须是矢量 PDF(能用鼠标选中文字的那种,一般由 CAD / 天正导出)。扫描件、拍照图不行。
结果存到留空 = 存到原 PDF 所在文件夹
第几页页码从 0 开始数
清晰度(DPI)默认 200 够用;栅格小、字小可调到 300
图纸类型hall(站厅)或 site(总平面)
启用 OCR勾上读图上数字(需要装 tesseract 系统级 OCR)

输出(在选定目录下):

  • 底图.jpg:从 PDF 渲染出的图片
  • 底图_prelabel.xml:CVAT 1.1 格式预标注(导入 CVAT 复审)
  • 底图_overlay.jpg:预标注效果叠加预览

2.2 质检(QC) qc

用途:检查 CVAT 标注文件是否符合 schema 规范。能自动找出:

  • 🔴 ERROR:非法标签、退化几何(零面积)、必现类缺失(如总平面图必须有 surrounding_building)
  • 🟠 WARN:属性值不在枚举集、数量异常(商铺 > 3 个)、宽度尺寸线两端无对应实体(ORPHAN)
  • 🔵 INFO:重复重叠、属性全留空提示(可能上游有文字层未代抽)

输入:标注 XML 文件 + 图纸类型(站厅 / 总平面 / 自动判断)

输出:在日志区显示分级报告。ERROR 数 = 0 才可交付。

2.3 转换 convert

用途:把 CVAT 里的中文标签批量改成英文 snake_case(规范要求 schema 100% 英文)。

特性:带"漏转自检" — 没在映射表里的中文标签会报错并列出,不会静默漏掉。

新批次工作流(推荐):CVAT 项目按 rules/cvat_labels.json 锁死下拉菜单后, 标注员选的就是英文标签,convert 这一步几乎不再需要。具体见 docs/guides/cvat_template_setup.md。

3. 页面二:端到端预审 主功能

这是 MVP 的核心功能 — 拖入 PDF → 一键审图,输出标注图 + 结构化数据 + Word 审查意见表。

3.1 4 行输入

字段必填?说明
图纸 PDF是矢量 PDF(能选中文字)。扫描件/拍照图不行
方式① CVAT 真值标注 (.xml)否已有 CVAT 标注 XML → 走真值路径,识别 100% 准
方式② YOLO 模型权重 (.pt)否best.pt → 切片推理转 CVAT 喂下游,零人工
结果存到否留空 = PDF 旁新建 e2e_out/

3.2 数据源决策(自动 4 路径)

点 开始预审 后,客户端按优先级选数据源:

优先级路径触发条件说明
① 最优方式① CVAT 真值用户填了 CVAT XML 字段真值标注 → 识别 100% 准
② 推荐方式② YOLO 自动识别用户填了 .pt 权重字段切片推理 → 转 CVAT XML → 走全套规则引擎
③ 自动自动匹配 CVAT用户两个都没填按文件名站名+图类型在 data/anno_*/ 找 XML,命中即走方式①
④ 兜底旧 e2e_demo OCR都没匹配到只查防火分区面积,精度低
最常用工作流:留空方式①方式② → 自动匹配站名(嘉宾站、欧洲城站、明海等已知站 ✅; 文件名无站名时退回兜底)。想强制走 YOLO 就填权重。

3.3 实时阶段进度

左侧"流程进度"显示 4 个阶段(真值路径):

阶段做什么
[0/4] 比例尺标定读 PDF 矢量文字层 "1:N" 比例 + jpg 大小算 m/px
[1/4] CVAT 标注 → 结构化对象适配器派生 fire_compartment / door / exit 等 12 类对象
[2/4] 规则引擎评估跑 37 条规则,输出 PASS/FAIL/REVIEW
[3/4] 在底图标注违规位置红框标 FAIL + 右上角"全局违规"徽章(无坐标的违规也能看见)

3.4 产物(2026-06 新命名)

产物前缀按"站名-图类型"自动命名(如 嘉宾站-站厅层_xxx),不再用设计院图号:

文件大小用途
<站>-<图类型>_e2e_fail.jpg~3-4 MB底图 + 红框标注违规位置 + 全局违规徽章
<站>-<图类型>_findings.json~60 KB规则引擎原始评估结果(程序消费)
<站>-<图类型>_structured.json~20 KB结构化对象(规则引擎输入)
<站>-<图类型>_report.md~2 KBMarkdown 简报(grep "FAIL:" 批量看)
<站>-<图类型>_审查意见表.docx ⭐~3 MB可交付 Word 报告,含标准审查表 + 嵌入标注图 + 复核情况(□通过 □不通过)

YOLO 路径额外多一个 <站>-<图类型>_yolo_auto.xml(中间产物,YOLO 推理转出的 CVAT)。

3.5 导出 Word 报告

预审跑完后,导出 Word 报告 按钮直接用 Word 打开 <站>-<图类型>_审查意见表.docx — docx 已在端到端流程结尾自动生成,不再单跑脚本。

3.6 标注图怎么看

  • ■ 红框 = 不合规违规位置(对应"图上编号"列)
  • ■ 黄框 = 待人工复核(数据不足或属性 unknown)
  • 右上角红色徽章 = 没有具体坐标的"全局违规"(如"公共区出口数 0 < 2"),target=station 全局对象
看不见红框?有可能是这张图全是"全局违规"(target=station 没坐标),所有标注都进了右上角徽章 — 往图右上角看(不是图正中)。

5. 页面四:YOLO 训练 高级

面向懂模型的人 — 保留 epochs/imgsz 等术语,旁边都配了说明。流程分三步:

5.1 第一步:建数据集

参数说明默认
标注目录里面要有 CVAT XML 和对应底图,成对存放(支持 images/default/ 嵌套)—
数据集存到输出 YOLO-seg 格式的训练集(含 data.yaml)—
切片大小大图切小块训练,边长 px;填 0 = 不切1024
切片重叠块之间留点重叠,免得目标被切断128

跑完后会自动把 data.yaml 衔接到下一步的"训练 / 评估"页。

5.2 第二步:训练

参数说明建议
数据集配置上一步生成的 data.yaml—
预训练权重从这个权重接着学,别从零训yolo11s-seg.pt
epochs训练轮数,越多越久100(先试) / 50(快)

5.3 第三步:评估

  • 选 data.yaml + 训练产出的 best.pt
  • 看 mAP50(整体准确度)和每类的 P/R(查准率/查全率)
  • 混淆矩阵图会显示在右上方,看哪类元素容易认错
实测基准(2026-06 数据 18 张 / RTX 5060 / yolo11s-seg / 37 epoch): gate=0.995 / vent_group_ground=0.98 / fire_door=0.92 / station_exit_ground=0.79; 样本少的类(safety_exit=4 实例 / commercial_shop=1 实例)mAP=0 — 这是数据量问题, 后续按 docs/guides/yolo_training_guide.md 增量加数据即可。

6. 常见问题

Q1. 客户端卡住怎么办?

所有耗时操作都是子进程,理论上不会卡界面。如果真卡住,点 停止; 最坏可以直接关窗口,数据安全(不会丢已写的文件)。

Q2. PDF 用不了?

预标注/端到端必须是矢量 PDF(能选中文字)。扫描件 / 拍照图 = 栅格 PDF, 会被识别为"无文字层"并降级为人工标注模式。

Q3. 训练页"首次会自动下载预训练权重",但下载卡住?

预训练权重 (yolo11s-seg.pt) 20MB 左右,从 GitHub 拉。 网络不通时手动下载放项目根,客户端会优先用本地的。

Q4. 端到端预审报 "❌ 未检测到 tesseract(OCR 引擎)"?

Python 装的 pytesseract 只是封装,真正的 OCR 引擎是系统级程序需要单独装。 最常见的 Windows 解决步骤:

  1. 下载 UB-Mannheim/tesseract 安装包
  2. 安装时勾选 "Additional script data → Chinese Simplified"(中文图纸需要)
  3. 装到默认路径(C:\Program Files\Tesseract-OCR\)即可 — 项目脚本会自动探测到这里
  4. 重启客户端 → 端到端预审重新跑,会自动找到

如果还是报"未检测到"(说明装到了非默认路径):

  1. 找到实际的 tesseract.exe 路径(右键开始菜单中的 Tesseract 图标 → 打开文件位置)
  2. 把所在文件夹加入系统 PATH 环境变量(Win+R → sysdm.cpl → 高级 → 环境变量 → 系统变量的 Path 编辑 → 新建 → 粘贴文件夹路径)
  3. 新开终端跑 tesseract --version 验证

macOS / Linux 安装命令见 §1.2 系统级依赖 — Tesseract OCR。

矢量 PDF 不一定需要装 — 文字层直读优先;只有无文字层时才必须有 tesseract。
真实案例(本项目机器实测):Tesseract 装在 C:\Program Files\Tesseract-OCR\ 但 PATH 没设 → tesseract --version 命令行报"不是内部或外部命令" → 但项目脚本通过 tesseract_init.ensure_tesseract() 自动探测到了, OCR 链路正常工作。所以"命令行找不到"不等于"项目跑不了"。

Q4. 字体显示不全 / 中文乱码?

Windows 默认字体即可。Linux 需要装思源黑体或微软雅黑; 日志输出乱码可设 $env:PYTHONUTF8=1。

Q5. 端到端预审跑得不准?

第一版的真实精度受两件事制约:

  1. YOLO 模型数据量:某些类(商铺、防火卷帘)训练实例 < 20 个,mAP 偏低
  2. 比例尺标定:目前从 PDF 矢量文字层读"1:N";栅格 PDF 无法自动标定

修正:补标新站点数据 + 重新训练(走 docs/guides/yolo_training_guide.md 增量流程)。
